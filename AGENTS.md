# Agent Platform — AI Agent Guide

This file provides context for AI coding agents (GitHub Copilot, Codex, etc.) working on this codebase.

## Project Overview

Kubernetes operator that implements budget-aware agent scheduling with pool-based fallback. Built with kubebuilder + controller-runtime.

**Domain:** `agents.platform/v1`
**Module:** `github.com/ezequiel/agent-platform`

## Key Concepts

- **Agent**: executor with a pool (team/shared/global), skills, and a budget reference
- **Skill**: capability with a token cost
- **Budget**: spending limit tracked in-memory (mock for Redis)
- **Task**: work request matched to an agent via the scheduler
- **Pool fallback**: team → shared → global priority order

## Architecture

```
Task CR created → TaskController.Reconcile()
  → scheduler.Schedule(task)
    → findAgentsBySkill(skill)
    → sort by pool priority (team=0, shared=1, global=2)
    → for each agent: hasBudget? → deduct & return
    → if none: return error
  → update Task status (phase, assignedAgent, reason)
  → update Budget status (used += cost)
```

All three controllers share a single `*scheduler.Scheduler` instance injected via `cmd/main.go`.

## How the Scheduler Works

The scheduler is in `scheduler/scheduler.go`. Key methods:

| Method | Purpose |
|--------|---------|
| `RegisterAgent(agent)` | Add/update agent in registry |
| `UnregisterAgent(name)` | Remove agent |
| `SyncBudget(budget)` | Update budget remaining from CR |
| `Schedule(task)` | Find agent by skill → check budget → fallback → return result |

Budget state is **in-memory** (maps). The `BudgetController` and `TaskController` keep it in sync with the actual Budget CRs.

## Common Tasks

### Add a new Skill
1. Create YAML in `manifests/skills/`
2. Reference it in an Agent's `spec.skills` list

### Add a new Agent
1. Create YAML in `manifests/agents/` with pool, skills, budgetRef
2. Create corresponding Budget in `manifests/budgets/`

### Add a new pool type
1. Add to `PoolPriority` map in `scheduler/scheduler.go`
2. Add to `+kubebuilder:validation:Enum` in `api/v1/agent_types.go` and `api/v1/budget_types.go`
3. Run `make generate manifests`

### Modify CRD fields
1. Edit `api/v1/*_types.go`
2. Run `make generate manifests`
3. Update controllers if needed
4. Update sample manifests

Test scenarios covered:
- Happy path scheduling
- Budget exhaustion with fallback
- No budget available (error)
